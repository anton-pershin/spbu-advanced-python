# Практика 2: Классы, состояние и поведение (LRU cache)

Эта практика знакомит с классами в Python как с инструментом проектирования небольших, понятных API вокруг *изменяемого состояния* (mutable state).

В качестве структуры данных мы реализуем **LRU cache** (Least Recently Used): ограниченное по размеру хранилище "ключ -> значение", которое при переполнении удаляет *наименее недавно использованный* элемент.

## Цели

К концу практики вы должны уметь:

- создавать классы: `class`, `__init__`, атрибуты экземпляра, методы
- проектировать публичный API (что доступно пользователю) и скрывать детали реализации
- делать удобное представление объектов: `__repr__` (для отладки) и `__str__` (для человека)
- поддерживать инварианты структуры данных ("объект всегда в согласованном состоянии")
- реализовать LRU cache так, чтобы `get` и `put` работали за **O(1)** в среднем

## Класс и объект

*Класс* описывает структуру и поведение. *Объект* (экземпляр) - конкретное значение, созданное по этому описанию.

```python
class Point:
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y

p = Point(1.0, 2.0)
```

- `__init__` инициализирует *уже созданный* объект. Сам объект создается вызовом `__new__`, который Python делает за нас.
- `self` - это экземпляр; по соглашению это первый аргумент всех методов экземпляра.

### Атрибуты экземпляра и `__dict__`

По умолчанию атрибуты экземпляра хранятся в `obj.__dict__`:

```python
p.__dict__  # {'x': 1.0, 'y': 2.0}
```

Дополнительно к этому существует, `__slots__`, который позволяет ограничить набор атрибутов и может существенно уменьшить память для множества маленьких объектов.

### Public/private/protected аттрибуты

Строго говоря, в Python все аттрибуты класса являются public аттрибутами. Чтобы подчеркнуть приватность аттрибута, используют нижнее подчеркивание в начале названия аттрибута: `_attrname`. Python также предоставляет более безопасный способ задания private аттрибутов с целью защиты от пересечения названий переменных с базоывми классами, от которых происходит наследование. Этот способ заключается в добавлении двух подчеркиваний в начале названия аттрибута: `__attrname`. Python использует name mangling и преобразует это название в `_classname__attrname`.

### Специальные методы

#### Создание экземпляра класса: `__new__`

Метод `__new__` является статистическим методом класса, который принимает на вход класс и создает его экземпляр. Обычно он вызывает `super().__new__(cls, ...)`.

#### Инициализация экземпляра класса: `__init__`

Метод `__init__` вызывается сразу после `__new__` и наполняет уже созданный экземляр.

#### Подготовка к удалению экземпляра: `__del__`

Метод `__del__` выполняется перед удалением экземпляра и позволяет подготовиться к этому удалению (например, закрыть соединения). Удалением экземпляра из памяти произойдет при обнулении счетчика ссылок на уровне сборщика мусора.

#### Строковая репрезентация в интерпретаторе: `__repr__`

Метод `__repr__` возвращает строковое представление экземпляра, как оно появляется в интерпретаторе.

#### Строковая репрезентация в остальных случаях: `__str__`

Метод `__str__` возвращает строковое представление экземпляра, используемое при вызове `str()` и `print()`

## Что такое LRU cache

LRU cache поддерживает две основные операции:

- `get(key)` — вернуть значение по ключу и отметить ключ как *самый недавно использованный* (MRU)
- `put(key, value)` — добавить/обновить значение и отметить ключ как *самый недавно использованный* (MRU)

У кеша есть фиксированная емкость `capacity`. Если после `put` элементов стало больше, чем `capacity`, нужно удалить ровно один элемент — **наименее недавно использованный** (LRU).

### Почему одного `dict` недостаточно

Словарь дает O(1) доступ по ключу, но для LRU нужно еще уметь быстро:

- менять "свежесть" ключа при каждом `get`/`put`
- удалять именно "самый старый" ключ

Классическое решение использует две структуры данных:

- `dict[key] -> node` для O(1) доступа к узлу
- **двусвязный список** для поддержания порядка по "свежести":
  - начало списка — MRU
  - конец списка — LRU

Тогда мы можем:

- перемещать произвольный узел в начало за O(1)
- удалять узел с конца за O(1)

### Задание 2. Реализация `LRUCache`

Реализуйте `LRUCache` со следующим публичным API:

- `LRUCache(capacity: int)`
- `get(key, default=None)`
- `put(key, value)`
- `items()` -> список пар `(key, value)` от MRU к LRU (используется в проверках и удобно для отладки)
- `__len__` и `__contains__`
- `__repr__`

Ограничения:

- `capacity` — положительное целое число
- `get` и `put` должны работать за O(1) в среднем
- инварианты должны сохраняться всегда (согласованность словаря и списка)
