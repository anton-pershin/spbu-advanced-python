# Practicum 1: Modules, Packages, Project Structure

Практика посвящена базовым инженерным навыкам работы с Python-проектами:
- что такое модуль и пакет
- как работает импорт и почему важно запускать код из корня проекта
- как сделать небольшой пакет с "публичным API"
- как сделать CLI-точку входа через `python -m ...`

## Цель

Дан "монолитный" скрипт, который считает частоты слов в текстовых файлах. Ваша задача — разнести код по модулям и оформить его как пакет.

К концу практики у вас должны быть:
- пакет `textstats/` с несколькими модулями
- CLI-запуск через `python -m ...`
- минимальная проверка корректности через `check.py` (без `pytest`)

## Как запускать (важно)

Мы запускаем всё **из корня репозитория**.

Примеры:

1) Запуск CLI:
```bash
python -m p_01_modules_packages.template.textstats p_01_modules_packages/template/data/sample.txt --top 10
```

2) Запуск проверок:
```bash
python -m p_01_modules_packages.template.check
```

Почему так:
- Python строит `sys.path` относительно текущей директории
- при запуске из корня проще и стабильнее импортировать пакеты

## Структура заготовки

- `template/textstats/_monolith.py` — исходный скрипт ("до")
- `template/textstats/` — пакет, который вы собираете ("после")
  - `io.py` — чтение текстов
  - `normalize.py` — токенизация и нормализация
  - `stats.py` — подсчет и сортировка
  - `cli.py` — разбор аргументов и печать результата
  - `__init__.py` — публичный API
  - `__main__.py` — запуск `python -m ...`

## Что делает `__init__.py`

Файл `__init__.py` делает директорию **пакетом** (package) и выполняется при импорте пакета.

В этой практике `__init__.py` нужен по двум причинам:
- чтобы Python мог импортировать модули по длинному пути `p_01_modules_packages.template.textstats...` при запуске из корня репозитория
- чтобы собрать "публичный API": например, чтобы работал импорт
  ```python
  from p_01_modules_packages.template.textstats import tokenize, count_words, top_n
  ```
  даже если реальные реализации лежат в `normalize.py` и `stats.py`

Мини-задание (эксперимент):
1) Временно переименуйте `p_01_modules_packages/template/textstats/__init__.py` (например, в `__init__.py.bak`).
2) Запустите проверку из корня:
   ```bash
   python -m p_01_modules_packages.template.check
   ```
3) Посмотрите на ошибку импорта и объясните, почему она произошла.
4) Верните файл обратно.

## Задания

### Задание 0. Понять ожидаемое поведение

Запустите монолитный скрипт и посмотрите на формат вывода:
```bash
python -m p_01_modules_packages.template.textstats._monolith p_01_modules_packages/template/data/sample.txt --top 10
```

### Задание 1. Разбить код на модули

Перенесите логику из `_monolith.py` в модули пакета:
- чтение файлов -> `io.py`
- разбиение текста на токены -> `normalize.py`
- подсчет частот и top-N -> `stats.py`

### Задание 2. Собрать публичный API

В `textstats/__init__.py` экспортируйте только то, что считается публичным:
- `tokenize`
- `count_words`
- `top_n`

Внутренние детали (`io`, `cli`) не должны быть обязательны для импортирующего кода.

### Задание 3. Сделать CLI через `python -m ...`

Реализуйте `textstats/cli.py` и `textstats/__main__.py` так, чтобы работало:
```bash
python -m p_01_modules_packages.template.textstats <paths...> --top 10
```

Требования к CLI:
- принимает один или несколько путей к файлам
- флаг `--top N` (по умолчанию 10)
- флаг `--lower/--no-lower` (по умолчанию `--lower`)
- печатает строки `word<TAB>count`, отсортированные по убыванию count, затем по алфавиту

### Задание 4. Проверка

Запустите:
```bash
python -m p_01_modules_packages.template.check
```

Если проверки падают — исправьте реализацию.

## Частые ошибки

- Запуск `python p_01_modules_packages/template/textstats/cli.py` (так ломаются импорты). Используйте `python -m ...`.
- Смешивание CLI и логики: CLI должен только разобрать аргументы и вызвать функции.
- Неустойчивая токенизация: договоритесь о правилах и реализуйте их строго.

## Подсказка по токенизации

Для простоты считаем "словом" последовательность латинских/кириллических букв или цифр. Остальное — разделители.

Подойдет регулярное выражение `r"[0-9A-Za-zА-Яа-яЁё]+"`.
