# spbu-advanced-python

Дополнительные материалы по продвинутому Python для практик по дисциплине "Основы алгоритмов", читаемой на факультете ПМ-ПУ в СпбГУ.

## Getting started

Для начала необходимо настроить свой python environment. Рекомендуется использовать [miniconda](https://docs.anaconda.com/miniconda) и python 3.10 и выше. Вам также необходимо будет скачать последние версии numpy, matplotlib и других библиотек, перечисленных в `requirements.txt`. Находясь в ваш python environment, вы можете это сделать через
```bash
cd /path/to/your/repo/clone
pip install -r requirements.txt
```

Работая с данным репозиторием, вам необходимо будет экспортировать функции из него и запускать скрипты из корня репозитория. Следовательно, вам необходимо добавить его в PYTHONPATH. Ниже перечислены несколько способов, как можно добавить корень репозитория в PYTHONPATH.

### Terminal (python venv и conda)

Выполните следующую команду в терминале и продолжайте работу в той же сессии:
```bash
export PYTHONPATH=/path/to/your/repo/clone:$PYTHONPATH
```
Эта команда должна выполняться каждый раз, когда создается новая сессия. Чтобы упростить себе жизнь, можно добавить вызов это команды в конфиг вашего терминала/шелла (например, `.bash_profile` или `.bash_rc`.

### Terminal (только conda)

Выполните следующую команду в терминале с активированным conda environment:
```bash
conda develop /path/to/your/repo/clone
```

### VSCode

Выберите `Run and Debug` в левой панели, создайте новый launch file, выберите `Python File` и добавьте следующее поле:
```yaml
"env": {
    "PYTHONPATH": "${workspaceFolder}${pathSeparator}${env:PYTHONPATH}"
}
```

## Структура репозитория

Организация материалов:
- каждая практика соответствует отдельной директории `p_XX_short_name/`
- в каждом практикуме: `p_XX_short_name/README.md` (теория + постановка задач) и один или несколько шаблонов в отдельной директории `p_XX_short_name/template/*.py` с блоками
  ```python
  ##########################
  ### PUT YOUR CODE HERE ###
  ##########################
  ```
  и соответствующие им решения в директории `p_XX_short_name/solution/`

Ожидаемые инструменты качества к концу курса:
- `pytest` (тесты)
- `black`, `isort` (форматирование и импорты)
- `mypy` (проверка типов)
- `pylint` (линтер)
- упаковка проекта через `pyproject.toml` и установка в editable-режиме (`pip install -e .`)

## План практик

1. Модули, пакеты, структура проекта
   - imports, модули/пакеты, `__name__ == "__main__"`, простая CLI-точка входа
   - рефакторинг монолитного скрипта на несколько модулей

2. Классы: состояние и поведение
   - `class`, `__init__`, `__new__`, методы, `__repr__`/`__str__`, проектирование API
   - внутреннее устройство классов
   - переход от "набор функций" к объектам там, где это действительно упрощает код

3. Наследование и композиция
   - `super()`, Method Resolution Order (MRO), переопределение методов, когда наследование уместно
   - Интерфейсы через абстрактные классы: `abc.ABC`, `@abstractmethod`
   - композиция как альтернатива; рефакторинг "плохой" иерархии

4. Модели данных и валидация: `property`, `dataclass`, `TypedDict`, Pydantic
   - внутренние модели: инварианты, свойства, `@dataclass`
   - внешние данные: `TypedDict`, Pydantic-модели, валидация и преобразование входных данных

5. Функции как объекты + функциональные приемы
   - функции высших порядков, `Callable`, `lambda`, `sorted(key=...)`, `any/all`, `map/filter`, `functools.reduce` (и альтернативы через comprehension/обычные циклы там, где они читаемее)
   - `functools.partial` (как практичный инструмент конфигурирования)
   - сборка простого "конвейера" преобразования данных из функций

6. Замыкания и декораторы (`functools`)
   - замыкания, аккуратное использование `nonlocal`
   - декораторы, `functools.wraps`
   - практические декораторы: таймер, простое кеширование, валидация аргументов

7. Итераторы, генераторы (ленивые вычисления) и протоколы
   - протокол итератора, генераторные функции/выражения, `yield from`
   - `itertools` как набор готовых кирпичиков для ленивой обработки последовательностей
   - ленивые пайплайны обработки данных
   - утилиты вида `take`, `chunked`, потоковая обработка файлов
   - `typing.Protocol` как способ описывать структурные интерфейсы для источников/потребителей данных

8. Качество кода и упаковка проекта (`pyproject.toml`)
   - структура `src/`, установка `pip install -e .`
   - конфиги `pytest/black/isort/mypy/pylint` в `pyproject.toml`
   - единый "quality gate": набор команд, который прогоняется перед сдачей
